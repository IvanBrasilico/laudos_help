Detalhes adicionais da implementação de segurança.
TODO acesso é validado. O sistema lê o usuário atualmente "logado" no ambiente Windows e checa se está no Banco de Dados. Em seguida, checa o perfil deste usuário e consulta uma lista de controle de acesso.

Nesta lista de controle de acesso, o Acesso pode ser dado(ou não) para a abertura de uma tela ou para o acesso ao Banco de Dados, na modalidade leitura ou modificação, no caso do Banco de Dados. Tanto a parte do sistema (classe) que faz acesso ao Banco de Dados quanto a parte do sistema (classe) que exibe as telas checam este bando de "ACL" antes de permitirem a operação. A falta de permissões interrompe uma ação do sistema e exibe uma mensagem de erro.
